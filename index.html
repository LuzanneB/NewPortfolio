  • add close9/26/2019 Not everything requires a function

    The bootcamp I just completed focuses so much on Javascript my initial approach to almost everything is to write a function, especially when a click event is involved. Today as I was writing an onClick function to toggle the open and close icons for this collapsible, when it dawned on me that there must be a better way. The JQuery that I had put together (similar to what my team used when we put together Food Decoder, linked in the Code section of this page) was slow on the page and was going to require even more code to make it run properly. “It’s just an icon”, I thought.

    After a very quick google search, I found that my initial instinct to write a function was overcomplicating it. The same way I was tapping into the “active” class on the li for a JQuery function to add and remove the class for the corresponding icons, I could put in some super simple CSS to display or not display. What else am I overcomplicating?! The more I learn about CSS, the more magical, simple and fast I find that it is.
